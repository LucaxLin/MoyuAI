# 墨语 (MoyuAI) 项目设计文档

## 一、项目概述

### 1.1 项目背景

墨语是一款基于人工智能的创意图片生成应用，用户可以通过自然语言描述或上传参考图片，快速生成高质量的AI艺术作品。应用采用对话式交互设计，让图片创作过程更加直观和便捷。核心技术基于MiniMax image-01模型，支持文本转图片、图片编辑、图片风格迁移等多种创作模式。

随着AI技术的快速发展，AI图片生成已经成为内容创作、设计工作、广告营销等领域的重要工具。墨语致力于为普通用户提供简单易用的AI创作体验，降低AI图片生成的门槛，让每个人都能成为创意设计师。

### 1.2 项目目标

墨语项目的核心目标是打造一款**轻量级、易用性强、设计精美的AI图片生成应用**。具体目标包括：

**用户体验目标**：采用对话式交互设计，让用户像聊天一样进行图片创作。界面设计追求现代简洁风格，摒弃繁杂的边框和装饰元素，突出内容本身。应用需完美适配PC端和移动端，并支持明暗主题切换，满足不同场景的使用需求。

**功能目标**：实现完整的用户体系管理，包括邮箱验证码注册、邮箱密码登录。提供文本生图、图片+文本生图、图片编辑三大核心功能。图片编辑支持智能识别用户意图（基于原图修改或重新生成）、关键词触发二次创作、框选区域局部修改等高级特性。构建用户私有图库，支持图片收藏、下载、引用二次创作。

**技术目标**：采用现代化的Monorepo架构，使用Turborepo + pnpm workspaces管理多包项目，确保应用具备良好的可维护性、可扩展性和性能表现。前端采用Next.js 15 (App Router) + shadcn/ui组件库，后端使用Next.js API Routes（Serverless Functions），实现前后端统一部署到Vercel平台。数据存储采用Vercel Postgres（托管PostgreSQL），图片存储采用Vercel Blob对象存储，实现零运维数据库和存储方案。

### 1.3 核心价值

墨语的核心价值体现在以下三个维度：

**创作赋能**：通过AI技术赋能创意表达，用户无需具备专业的设计技能或绘画功底，只需用自然语言描述脑海中的画面，AI即可将其转化为视觉作品。这极大地降低了创意表达的门槛，让更多人能够实现自己的创意想法。

**交互创新**：区别于传统的图片生成工具，墨语采用对话式交互设计。用户在创作过程中可以与AI进行多轮对话，逐步完善和调整生成结果。系统智能识别用户的编辑意图，自动判断是基于当前结果继续优化，还是回到原图重新生成。

**隐私保护**：图库功能采用严格的隐私隔离设计，用户只能查看和操作自己收藏的图片，他人的作品完全不可见。这种设计既保护了用户的创作隐私，也避免了潜在的版权和知识产权纠纷。

### 2.2 技术栈

#### Monorepo管理工具

**包管理器：pnpm**

选用理由：pnpm是高性能的包管理器，采用硬链接技术大幅减少磁盘空间占用。pnpm workspaces原生支持Monorepo项目管理，可以优雅地管理多个包的依赖关系。相比npm和yarn，pnpm在依赖安装速度和磁盘空间利用率上都有显著优势。

**构建工具：Turborepo**

选用理由：Turborepo是Vercel开源的高性能Monorepo构建工具，支持增量构建、智能缓存和并行执行。项目中的前端和后端可以独立构建和部署，同时共享构建缓存提高开发效率。相比Nx，Turborepo配置更简洁，与Next.js生态系统深度集成。

#### 前端技术栈 (apps/web)

**框架：Next.js 15 (App Router)**

选用理由：Next.js 15是React生态中最成熟的全栈框架，App Router支持服务端渲染(SSR)和客户端渲染(CSR)混合模式。使用Next.js API Routes作为后端接口，统一部署到Vercel平台，实现前后端一体化开发。

**UI组件库：shadcn/ui + Tailwind CSS**

选用理由：shadcn/ui是2024年最流行的React UI组件库，采用headless设计理念，组件样式完全由Tailwind CSS控制，可实现高度定制化。符合项目"现代简洁、无边框"的UI设计要求。

**状态管理：Zustand**

选用理由：Zustand是轻量级状态管理库，API简洁，体积小(约1KB)，非常适合Next.js应用。相比Redux，Zustand无需编写大量模板代码，支持React Hooks语法，学习成本低。项目主要的状态管理需求集中在前端(UI状态、主题、用户输入)，Zustand完全满足需求。

**路由管理：Next.js App Router**

Next.js内置的App Router提供基于文件系统的路由约定，支持布局嵌套、加载状态、错误边界等高级功能。客户端导航使用 `<Link>` 组件实现 SPA 无刷新跳转。

**HTTP客户端：原生 fetch API**

Next.js内置支持原生fetch API，配合Server Actions实现前后端数据交互。对于复杂请求，使用@tanstack/react-query进行数据获取和缓存管理。

**其他关键依赖**

- **@tanstack/react-query**：服务端状态管理，处理API请求缓存、重试、乐观更新
- **react-hot-toast**：轻量级Toast通知库
- **lucide-react**：图标库，与shadcn/ui完美兼容
- **date-fns**：日期处理库
- **next-themes**：主题管理，支持明暗主题切换
- **@vercel/blob**：Vercel Blob文件存储SDK

#### 后端技术栈 (Next.js API Routes)

**运行环境：Vercel Serverless Functions**

Next.js API Routes基于Node.js环境，构建为Vercel Serverless Functions。Serverless架构自动扩缩容，无需管理服务器，降低运维成本。

**Web框架：Next.js API Routes**

Next.js API Routes基于Node.js HTTP服务器构建，部署到Vercel后自动转换为Serverless Functions。支持RESTful API开发，与前端共用同一个Next.js项目，降低技术栈复杂度。

**ORM：Prisma (@moyu/db)**

选用理由：Prisma是现代化的TypeScript ORM，支持强类型自动生成，提供优雅的API设计。Prisma Schema定义在@moyu/db包中，通过Prisma Client访问Vercel Postgres。相比传统Knex.js，Prisma的Prisma Client提供更好的开发体验和类型安全。

**数据库：Vercel Postgres**

选用理由：Vercel Postgres是Vercel官方提供的托管PostgreSQL数据库，与Vercel平台深度集成，提供零配置数据库解决方案。支持强一致性、内置连接 pooling（通过Neon/Supabase）、自动备份等特性。免费额度（Hobby Plan）：3个数据库项目，1GB存储，足够个人项目使用。

**认证：NextAuth.js v5**

选用理由：NextAuth.js是Next.js官方推荐的认证解决方案，与Next.js深度集成，支持多种认证Provider。使用JWT进行无状态认证，配合refresh token实现安全的登录状态管理。v5版本支持App Router，提供更好的类型安全和开发体验。

**其他关键依赖**

- **bcryptjs**：密码加密
- **nodemailer**：邮件发送(验证码)
- **sharp**：图片处理(压缩、格式转换)
- **uuid**：生成唯一标识符
- **zod**：运行时数据验证
- **@vercel/kv**：Vercel KV缓存（Redis），用于验证码存储和API限流

#### 共享包技术栈 (packages/)

**@moyu/shared - 共享工具库**

- **TypeScript类型定义**：前后端共享的DTO类型、接口定义
- **工具函数**：日期格式化、字符串处理、数据转换等
- **常量定义**：错误码、配置常量等
- **数据验证**：使用Zod定义验证Schema，供前后端复用

**@moyu/db - 数据库Schema包**

- **Prisma Schema定义**：数据模型、关联关系、索引
- **Prisma Client生成**：类型安全的数据库访问
- **数据库迁移脚本**：统一管理数据库版本

## 三、功能模块

### 3.1 用户模块

用户模块是整个应用的基础模块，负责用户的注册、登录、个人信息管理等功能。

#### 3.1.1 邮箱验证码注册

**功能描述**：新用户通过邮箱注册，系统向邮箱发送6位数字验证码，用户输入验证码完成注册。

**业务流程**：

1. 用户输入邮箱地址和密码，点击注册按钮
2. 前端校验邮箱格式和密码强度(至少8位，包含数字和字母)
3. 后端生成6位随机验证码，存入Verification表，设置10分钟过期时间
4. 后端使用Nodemailer发送验证码邮件
5. 用户输入验证码，后端验证验证码正确性
6. 验证通过后，创建User记录，密码使用bcrypt加密存储
7. 创建用户默认会话，跳转到主页

**技术实现**：

- 验证码存储在PostgreSQL Verification表，设置10分钟过期时间
- 同一邮箱5分钟内只能发送一次验证码
- 验证码错误3次后自动失效
- 邮件使用SMTP协议发送，支持QQ邮箱/网易邮箱/企业邮箱

#### 3.1.2 邮箱密码登录

**功能描述**：已注册用户通过邮箱和密码登录系统。

**业务流程**：

1. 用户输入邮箱和密码，点击登录按钮
2. 后端验证邮箱是否存在
3. 后端使用bcrypt验证密码是否正确
4. 验证通过后，生成JWT Token或使用NextAuth Session
5. 返回用户信息，跳转到主页

**技术实现**：

- 使用NextAuth.js处理认证流程
- Session存储在Cookie中，支持HTTP Only、Secure属性

2. 前端校验输入内容(非空、长度限制500字符)
3. 前端显示用户消息气泡，添加到消息列表
4. 前端显示"AI正在创作中..."加载状态
5. 后端接收用户输入，调用MiniMax image-01 API生成图片
6. 图片生成完成后，保存到本地存储，更新Image表
7. 返回图片URL给前端，前端更新消息列表显示图片
8. 图片下方显示操作按钮(收藏、下载、编辑)

**技术细节**：

- MiniMax image-01 API调用超时时间设置为60秒
- 图片默认尺寸：1024x1024(可扩展为多种比例)
- 图片格式：WebP(压缩率高，画质好)

3. 图片上传到服务器临时目录，返回临时URL
4. 用户输入修改指令(如"把这幅画改成水彩风格")
5. 前后端交互流程同文本生图，携带原图URL和修改指令
6. MiniMax API接收原图和指令，生成修改后的图片

**技术细节**：

- 图片大小限制：单张不超过10MB
- 移动端调用系统相机或相册选择
- 上传前进行客户端图片压缩，减少传输时间

#### 3.2.4 智能图片编辑

**功能描述**：系统智能识别用户的编辑意图，自动选择合适的处理方式。

**编辑意图识别规则**：

| 用户输入关键词 | 编辑意图 | 处理方式             |
| ------- | ---- | ---------------- |
| "不对"    | 重新生成 | 基于原图重新生成，不保留任何元素 |
| "重新生成"  | 重新生成 | 基于原图重新生成，不保留任何元素 |
| "再来一张"  | 重新生成 | 基于原图重新生成，不保留任何元素 |
| "稍微调整"  | 微调   | 基于上次生成结果做细节调整    |
| "再改改"   | 微调   | 基于上次生成结果做细节调整    |
| "保留这个"  | 继续创作 | 基于上次生成结果继续扩展     |
| "继续"    | 继续创作 | 基于上次生成结果继续扩展     |
| 其他输入    | 继续创作 | 基于上次生成结果进行修改     |

**业务流程**：

1. 用户输入编辑指令
2. 后端检测输入内容是否匹配编辑意图关键词
3. 根据匹配结果决定使用的原图：
   - 重新生成：使用用户上传的原图
   - 微调/继续创作：使用上次AI生成的图片
4. 调用MiniMax API生成图片
5. 返回结果给前端

#### 3.2.5 框选区域局部修改

7. 调用MiniMax API进行局部重绘

**技术实现**：

## 四、API 设计

### 4.1 API 设计原则

墨语后端采用RESTful API设计规范，遵循以下原则：

- **资源导向**：URL代表资源，使用名词复数形式
- **HTTP方法语义**：GET(查询)、POST(创建)、PUT(更新)、DELETE(删除)
- **统一响应格式**：所有API返回统一结构{success, data, error}
- **状态码规范**：200成功、400客户端错误、401未认证、500服务端错误
- **分页规范**：列表API支持page和limit参数，返回总页数和总条数

## 六、第三方服务集成

### 6.1 AI服务集成

#### MiniMax image-01 API

**服务说明**：MiniMax是国内领先的AI大模型提供商，image-01是其图片生成模型，支持高质量的文本转图片和图片编辑功能。

**接入配置**：

```javascript
// 环境变量配置
MINIMAX_API_KEY=your_api_key
MINIMAX_API_BASE=https://api.minimax.chat/v1
MINIMAX_GROUP_ID=your_group_id
```

**API调用封装**：

```javascript
// lib/minimax.ts
import { NextResponse } from 'next/server';

const MINIMAX_API_URL = 'https://api.minimax.chat/v1/image_generation';

export async function generateImage(params: {
  model: 'image-01';
  prompt: string;
  imageUrl?: string;
  maskImageUrl?: string;
  maskPrompt?: string;
  maskRegions?: Array<{x: number; y: number; width: number; height: number}>;
  width?: number;
  height?: number;
}) {
  const response = await fetch(MINIMAX_API_URL, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.MINIMAX_API_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      model: params.model,
      prompt: params.prompt,
      image_url: params.imageUrl,
      mask_image_url: params.maskImageUrl,
      mask_prompt: params.maskPrompt,
      mask_regions: params.maskRegions,
      width: params.width || 1024,
      height: params.height || 1024,
    }),
  });

  if (!response.ok) {
    throw new Error(`MiniMax API error: ${response.status}`);
  }

  return response.json();
}
```

### 6.2 邮件服务集成

#### SMTP邮件发送

**服务说明**：使用Nodemailer通过SMTP协议发送注册验证码邮件。

**支持配置**：

- QQ邮箱SMTP
- 网易邮箱SMTP
- 企业邮箱SMTP
- SendGrid/Resend等邮件API

**配置示例**：

```javascript
// lib/email.ts
import nodemailer from 'nodemailer';

const transporter = nodemailer.createTransport({
  host: process.env.SMTP_HOST,
  port: parseInt(process.env.SMTP_PORT || '587'),
  secure: false,
  auth: {
    user: process.env.SMTP_USER,
    pass: process.env.SMTP_PASS,
  },
});

export async function sendVerificationCode(email: string, code: string) {
  await transporter.sendMail({
    from: `"墨语 AI" <${process.env.SMTP_USER}>`,
    to: email,
    subject: '墨语 - 注册验证码',
    html: `
      <div style="font-family: sans-serif; max-width: 600px; margin: 0 auto;">
        <h2>欢迎注册墨语</h2>
        <p>您的注册验证码是：</p>
        <div style="font-size: 32px; font-weight: bold; letter-spacing: 8px; padding: 20px; background: #f5f5f5; text-align: center;">
          ${code}
        </div>
        <p style="color: #666; font-size: 14px;">验证码有效期10分钟，请尽快完成验证。</p>
      </div>
    `,
  });
}
```

2. **配置构建命令**：

```javascript

  "buildCommand": "cd ../.. && pnpm --filter @moyu/web build",
  "outputDirectory": ".next",
  "installCommand": "cd ../.. && pnpm install",
```

### 8.3 Vercel Postgres 数据库

1. 在 Vercel Dashboard 中，点击 "Storage" → "Create Database"

3. 选择区域（推荐：hkg1 - 香港，sin1 - 新加坡）

```typescript
datasource db {

  // postgresql://user:password@host/database?sslmode=require
```

**Vercel Postgres 特点**：

### 8.4 Vercel Blob 文件存储

**创建存储桶**：

1. 在 Vercel Dashboard 中，点击 "Storage" → "Create Database"
2. 选择 "Blob"
3. 配置存储桶名称和权限

**使用示例**：

```typescript
// apps/web/src/lib/blob.ts
import { put, del, list } from '@vercel/blob';

export async function uploadImage(file: File, userId: string) {
  const filename = `${userId}/${Date.now()}-${file.name}`;

  const blob = await put(filename, file, {
    access: 'public',
    token: process.env.BLOB_READ_WRITE_TOKEN,
  });

  return blob.url;
}

export async function deleteImage(url: string) {
  await del(url, {
    token: process.env.BLOB_READ_WRITE_TOKEN,
  });
}

export async function getUserImages(userId: string) {
  const { blobs } = await list({
    prefix: `${userId}/`,
    token: process.env.BLOB_READ_WRITE_TOKEN,
  });

  return blobs;
}
```

## 九、风险评估

### 9.1 技术风险

#### 风险1：MiniMax API服务稳定性

**风险描述**：项目完全依赖MiniMax image-01 API进行图片生成，如果API服务出现波动或不可用，将直接影响用户体验。

**影响程度**：高

**发生概率**：中

**应对策略**：

1. **服务降级方案**：当API调用失败时，向用户显示友好错误提示，提供"稍后重试"按钮
2. **缓存策略**：对于相同prompt的请求，启用服务端缓存(24小时有效期)，减少API调用次数
3. **多模型支持**：预留接口扩展能力，未来可接入其他AI服务(如Midjourney、Stable Diffusion)作为备选
4. **监控告警**：配置API调用失败率监控，当失败率超过阈值时发送告警通知

#### 风险2：图片存储扩展性问题

**风险描述**：采用服务器本地存储方案，随着用户量增长，图片文件增多，将面临存储空间不足、单服务器IO瓶颈等问题。

**影响程度**：中

**发生概率**：高(用户量增长后必然发生)

**应对策略**：

1. **近期方案**：在服务器挂载大容量数据盘，使用LVM逻辑卷管理，后期可在线扩容
2. **中期迁移**：预留七牛云/腾讯云COS对象存储迁移路径，当存储量超过500GB时启动迁移
3. **数据清理**：实现图片自动清理策略，删除超过6个月且未被收藏的图片
4. **压缩优化**：使用sharp库进行WebP格式转换，相较JPEG可减少30-50%存储空间

#### 风险3：PostgreSQL数据库性能

**风险描述**：随着用户量和数据增长，数据库查询性能可能下降，特别是图片列表查询和全文搜索场景。

**影响程度**：中

**发生概率**：中

**应对策略**：

1. **索引优化**：在高频查询字段(userId、sessionId、createdAt、isFavorite)建立复合索引
2. **分页优化**：使用游标分页(Cursor-based Pagination)替代传统OFFSET分页，提升大列表查询性能
3. **读写分离**：后期引入只读副本，分担查询压力
4. **定期维护**：配置pg\_stat\_statements监控慢查询，定期执行VACUUM清理

### 9.2 业务风险

#### 风险4：用户注册率低

**风险描述**：邮箱注册流程相对繁琐(需验证邮箱)，可能导致潜在用户流失，注册转化率低。

**影响程度**：中

**发生概率**：中

**应对策略**：

1. **流程优化**：简化注册流程，支持手机号一键登录作为备选方案
2. **引导优化**：在登录/注册页面展示应用核心功能截图和用户案例，提高注册动力
3. **社交登录**：后续迭代支持微信、Google等第三方登录，降低注册门槛
4. **游客模式**：考虑提供有限功能的游客模式，让用户体验后再引导注册

#### 风险5：AI生成内容合规风险

**风险描述**：用户可能使用AI生成不当内容(如色情、暴力、政治敏感等)，带来法律和品牌风险。

**影响程度**：高

**发生概率**：中

**应对策略**：

1. **内容审核**：在API调用前，使用敏感词过滤库检测用户输入
2. **Prompt限制**：对输入prompt进行长度限制和敏感词屏蔽
3. **用户协议**：明确用户协议，禁止生成违规内容，保留追究权利
4. **举报机制**：提供内容举报入口，用户可举报违规图片
5. **人工审核**：对被举报的图片进行人工审核确认

#### 风险6：服务器成本压力

**风险描述**：AI图片生成是计算密集型任务，API调用成本较高，随着用户增长，成本压力可能超过收入。

**影响程度**：中

**发生概率**：中

**应对策略**：

1. **计费系统**：预留计费接口扩展点，当用户量达到一定规模后启用计费
2. **免费额度**：新用户赠送一定额度(如10次生成)，体验后决定是否付费
3. **缓存复用**：对相同prompt启用缓存，减少重复API调用
4. **资源优化**：生成完成后及时释放内存，优化服务内存占用

### 9.3 运维风险

#### 风险7：服务可用性

**风险描述**：服务器或数据库出现故障，导致服务不可用，影响用户体验和留存。

**影响程度**：高

**发生概率**：低

**应对策略**：

1. **高可用架构**：腾讯云服务器配置自动快照和故障迁移
2. **数据库高可用**：使用RDS主备实例，自动切换
3. **健康检查**：配置服务健康检查接口，及时发现异常
4. **灾备方案**：制定详细的故障应急响应流程和恢复预案
5. **监控告警**：配置多维度监控(CPU、内存、磁盘、API响应时间)，异常时及时告警

#### 风险8：数据安全风险

**风险描述**：用户数据(邮箱、密码、图片)泄露或被攻击者窃取，带来法律责任和用户信任危机。

**影响程度**：高

**发生概率**：低

**应对策略**：

1. **密码加密**：使用bcrypt加密存储密码，加密强度不低于10轮
2. **传输安全**：全站强制HTTPS，API使用JWT Token认证
3. **权限控制**：数据库账号权限最小化，应用使用只读账号
4. **日志审计**：记录关键操作日志，定期审计
5. **定期更新**：及时更新依赖库，修复安全漏洞
